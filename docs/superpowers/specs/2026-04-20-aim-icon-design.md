# AIM Icon 设计说明

## 背景与目标

当前 `AIM` 缺少一个可以稳定承载产品识别的正式 Icon。现有方向已经明确：可以借用 `🎯` 的“目标收敛 / 命中执行”语义，但不能直接把系统 emoji 当作品牌图形使用。

本次设计的目标是定义一个统一的品牌图形原则，供后续 favicon、app icon、README 品牌展示位与其他基础品牌触点复用，避免不同场景各自演绎、语义漂移或视觉风格失控。

一句话定义如下：**AIM 的 Icon 不是 emoji 本身，而是一个以 `🎯` 为母题、表达目标收敛与执行命中的品牌图形。**

## 设计结论

`AIM` 的正式 Icon 应采用**几何化、产品化、去 emoji 化的靶心图形**。

建议图形骨架如下：

1. 以同心圆或近似同心结构表达“目标收敛”。
2. 以中心点或中心负形表达“命中点 / 目标核心”。
3. 可选地引入一条经过约束的方向性元素表达“执行命中”，但不能做成写实箭矢插入效果。
4. 整体优先形成单一稳定轮廓，保证小尺寸下仍能先被识别为品牌标记，而不是装饰插画。

该 Icon 的核心不是“靶子很像 emoji”，而是“一个足够克制、明确、可重复使用的目标型品牌符号”。

## 图形语义

图形语义应固定为以下三层，不应额外叠加复杂隐喻：

1. **目标**：靶心结构表达系统始终围绕明确目标推进，而不是发散探索。
2. **收敛**：由外向内的层级表达任务、决策与执行朝目标状态持续收敛。
3. **命中**：中心点或方向性元素表达执行结果不是停留在规划，而是追求真正落点与命中。

后续任何实现都不应把该图形解释成娱乐化的“玩游戏命中靶子”，而应服务于 `AIM` 作为执行编排、目标推进系统的产品定位。

## 视觉风格约束

1. 必须是矢量化几何图形，不得直接使用系统 emoji、emoji 描边临摹稿或拟物化插画。
2. 风格必须克制、稳定、产品化，避免夸张高饱和、多材质、高光、阴影、拟物纹理。
3. 造型应优先依赖圆、点、线、切角或负形等基础几何关系，而不是复杂细节。
4. 默认应支持单色或低色数表达；即使存在品牌主色，也必须能退化到单色版而不丢失识别。
5. 小尺寸下必须先保住外轮廓和中心识别，不能依赖细箭头、细描边或密集层次。
6. 不把图形做成过度可爱、卡通、游戏化或情绪化表情符号。

## 多场景适配原则

### favicon

1. 只保留最核心轮廓与中心命中关系。
2. 优先使用 16px 至 32px 仍可辨认的极简版本。
3. 不依赖文字，不保留脆弱小细节。

### app icon

1. 在系统图标容器内保持居中、稳重、留白充足。
2. 可以比 favicon 多一层结构或更完整的命中关系，但仍需避免拟物化。
3. 在浅色和深色背景下都应保持稳定识别。

### README

1. README 中使用的图形应与正式 Icon 同源，不得另做一套更花哨的展示版替代主标。
2. 如需与 `AIM` 字标并置，图形应承担主识别，字标承担命名确认。
3. README 展示图不应把 Icon 包装成海报式插画主视觉。

### 品牌位

1. 用于页头、导航、启动位或产品角标时，应优先使用标准版或单色版。
2. 不同品牌位之间只能调整尺寸、留白、颜色版本，不能频繁改骨架。
3. 品牌位中的 Icon 必须保持“目标收敛与命中”的同一语义，不得演化成普通圆形装饰。

## 非目标

1. 本次不直接产出最终 SVG、PNG、favicon 文件或 app icon 素材。
2. 本次不定义完整品牌色系统、字标系统或整套视觉识别规范。
3. 本次不扩展为 mascot、插画体系或营销视觉语言设计。
4. 本次不追求把 `🎯` 做到一眼像 emoji，而是明确与 emoji 拉开品牌化距离。

## 后续落地建议

1. 先基于本说明探索 2 到 3 个几何草图方向，比较是否更偏“纯靶心”、或“靶心加轻方向性命中元素”。
2. 在选定骨架后，优先验证 `16px` favicon、应用图标容器内展示、以及 README 横向品牌位三种最低必要场景。
3. 先完成单色版，再决定是否引入极少量品牌色，避免颜色先行掩盖图形问题。
4. 最终交付时应至少包含标准版、单色版、深浅背景适配版，以及最小尺寸使用建议。
