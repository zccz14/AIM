# Server 默认端口调整为 8192 设计说明

## 背景 / 问题

当前 `modules/api/src/server.ts` 仍将本地 Node 启动默认端口写死为 `3000`，与本次仓库期望基线不一致。同时，当前 live 文档中仍存在引导用户使用旧端口示例的内容，容易让本地启动说明与运行时默认行为脱节。

本次需要用最小改动把 Server 的默认端口统一调整为 `8192`，并同步修正文档中仍在描述旧默认端口的当前有效说明，同时保持 `PORT` 环境变量覆盖能力不变。

## 目标

1. 将 `modules/api` 的运行时默认端口从 `3000` 调整为 `8192`。
2. 保持 `PORT` 环境变量覆盖行为不变；显式传入 `PORT` 时，仍以环境变量值为准。
3. 更新当前 live 文档、示例或配置说明中仍指向旧默认端口的内容，使其与新默认值一致。
4. 将改动范围收敛在最小必要集合，不引入额外启动参数、兼容层或文档重写。

## 非目标

1. 不修改历史记录性质的文档，例如 `docs/superpowers/plans/*.md` 或其他仅用于存档的历史文件。
2. 不改变 `PORT` 解析逻辑、非法端口回退逻辑或启动入口结构。
3. 不新增新的环境变量、CLI 参数或多端口配置机制。
4. 不修改与端口默认值无关的 API 行为、OpenAPI 内容或部署流程。

## 影响范围

预计影响以下当前 live 文件：

1. `modules/api/src/server.ts`
2. `docs/api/README.md`

如在实现过程中发现其他当前有效文档、示例或配置仍明确声明旧默认端口，可在不触碰历史文档的前提下按同一原则一并修正；若只是提及自定义端口示例且不声称“默认值”，则不必扩大范围。

## 预期行为

### 1. 运行时行为

1. 未设置 `PORT` 时，`modules/api` 默认监听 `8192`。
2. 设置 `PORT` 时，服务继续监听该环境变量指定的端口。
3. 若 `PORT` 无法解析为有效数字，回退行为保持与当前实现一致，仅将默认回退值替换为 `8192`。

### 2. 文档行为

1. 当前仓库中的有效说明应把默认访问端口更新为 `8192`。
2. 若文档给出显式启动示例，示例端口应与默认行为一致，或清楚表明该值只是自定义覆盖示例。
3. 历史计划、历史设计记录不作为本次同步对象，避免破坏历史上下文。

## 实现约束

1. 采用最小修改路径，优先直接替换默认端口常量与相关 live 文档示例。
2. 不为了这次改动抽象新的配置层或帮助函数。
3. 实现必须保留现有 `PORT` 覆盖优先级与启动入口导出方式。

## 最小验证

1. 检查 `modules/api/src/server.ts`，确认默认端口常量已改为 `8192`，且 `PORT` 环境变量覆盖逻辑未被改坏。
2. 检查 `docs/api/README.md`，确认当前示例与访问说明不再指向旧默认端口。
3. 本次为小范围实现与文档对齐改动，优先执行最小必要验证，不要求额外扩展测试范围。
