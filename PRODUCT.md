# Product

## Register

product

## Users

AIM is used by the AIM Director: the decision maker or owner responsible for watching autonomous Manager, Coordinator, and Developer work converge from the current baseline toward a README-defined goal state. The Director is not primarily operating inside an IDE. They need to understand progress, blockers, task pool health, dependencies, and points where human clarification or adjustment is required.

## Product Purpose

AIM is a README-oriented autonomous software development and baseline iteration system. The web GUI exists to make baseline convergence observable: current baseline versus goal state, iteration progress, success rate, blockers, task pool, task dependencies, and human intervention points. Success means the Director can quickly decide whether the autonomous system is progressing, stalled, or asking for strategic input.

## Brand Personality

Calm, rigorous, authoritative. AIM should feel like a methodology control room for autonomous baseline convergence: dense evidence, quiet confidence, and precise intervention points without terminal noise.

## Anti-references

Avoid generic SaaS dashboard cliches, IDE or plugin terminal aesthetics, flashy decoration, documentation or Notion-style whiteboard surfaces, and hero-metric templates. The interface should not feel like a developer console, marketing page, or decorative analytics template.

## Design Principles

1. Keep the Director above the implementation noise: expose decisions, evidence, blockers, and clarification needs before low-level task mechanics.
2. Treat methodology as the product surface: structure the GUI around baseline review, task pool topology, feedback, and intervention readiness.
3. Prefer calm density over spectacle: compact evidence and readable hierarchy matter more than visual novelty.
4. Make state semantics explicit: progress, rejected feedback, dependencies, and missing closure cues must be easy to scan.
5. Preserve autonomous flow: interactions should clarify and adjust the system, not turn the GUI into an IDE replacement.

## Accessibility & Inclusion

Target WCAG AA. Maintain visible focus states, semantic navigation and regions, readable contrast in dark and light themes, reduced visual noise, and responsive layouts for laptop and external monitor use.
